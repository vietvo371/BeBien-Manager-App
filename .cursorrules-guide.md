# 🎯 Hướng Dẫn Sử Dụng Cursor với BeBien Manager

## 📋 Tổng Quan
File `.cursorrules` đã được cấu hình để Cursor AI hiểu sâu về dự án BeBienManager và code như một **Senior React Native Engineer**.

## 🚀 Cách Sử Dụng

### 1. Chat với Cursor (⌘ + L / Ctrl + L)
Để hỏi đáp và nhận tư vấn nhanh:

**Ví dụ câu hỏi tốt:**
```
"Tạo component OrderCard hiển thị thông tin đơn hàng với animation"
"Implement màn hình approval cho Manager theo workflow trong rules"
"Sửa LoginScreen để support biometric authentication"
"Optimize FlatList performance cho danh sách 1000+ orders"
```

**Ví dụ câu hỏi nên tránh:**
```
"Code cho tôi một cái gì đó"  // ❌ Quá mơ hồ
"Làm thế nào để dùng React Native?" // ❌ Quá chung chung
```

### 2. Composer Mode (⌘ + I / Ctrl + I)
Đây là **tính năng mạnh nhất** - Cursor sẽ tự động tạo/chỉnh sửa nhiều file cùng lúc:

**Ví dụ yêu cầu:**

#### 📱 Tạo màn hình mới
```
Tạo màn hình PendingOrdersScreen cho Manager với:
- Lấy danh sách pending orders từ API
- Hiển thị dạng FlatList với OrderCard component
- Có nút Approve và Reject
- Khi approve thành công, gửi socket notification tới Cashier
- Thêm vào MainTabNavigator
```

#### 🔄 Implement real-time feature
```
Implement order cancellation flow theo workflow:
1. Kitchen staff nhấn Cancel
2. Gửi socket event tới server
3. Server lưu pending approval
4. Manager nhận notification và approve
5. Socket notify Cashier (chỉ is_master = 0)
Tạo đầy đủ service, hook, và UI components
```

#### 🎨 Refactor component
```
Refactor LoginScreen:
- Tách form logic ra custom hook useLoginForm
- Tách validation ra separate file
- Optimize animations với Reanimated
- Add biometric authentication
- Follow coding standards trong .cursorrules
```

#### 🏪 Tạo Zustand store
```
Tạo Zustand store cho Orders với:
- State: orders, pendingOrders, selectedOrder
- Actions: fetchOrders, approveOrder, rejectOrder
- Persist vào AsyncStorage
- Type-safe với TypeScript
```

### 3. Inline Edit (⌘ + K / Ctrl + K)
Chỉnh sửa code trực tiếp trong editor:

**Select code → ⌘ + K → Gõ yêu cầu:**
```
"Add loading state and error handling"
"Optimize this FlatList with React.memo"
"Convert to TypeScript with proper types"
"Add i18n translations"
```

## 💡 Best Practices Khi Dùng Cursor

### ✅ DO - Nên làm

1. **Cụ thể và rõ ràng**
```
❌ "Tạo màn hình order"
✅ "Tạo OrderDetailScreen hiển thị order info, items list, total price, 
    status badge, có nút Cancel với confirmation dialog"
```

2. **Nhắc về workflow và rules**
```
✅ "Tạo approval flow theo workflow trong .cursorrules, 
    nhớ chỉ notify tới Cashier (is_master = 0)"
```

3. **Yêu cầu follow standards**
```
✅ "Tạo component OrderCard theo styling guidelines, 
    dùng theme colors, animation với Reanimated"
```

4. **Đề cập file liên quan**
```
✅ "Update MainTabNavigator để thêm OrdersScreen, 
    và tạo types trong order-api.types.ts"
```

### ❌ DON'T - Tránh làm

1. **Quá chung chung**
```
❌ "Code một cái login screen"
❌ "Làm feature order"
```

2. **Không context**
```
❌ "Tạo component Button"  // Button để làm gì? Style thế nào?
```

3. **Yêu cầu làm quá nhiều cùng lúc**
```
❌ "Tạo toàn bộ app từ đầu với auth, orders, chat, reports..."
```

## 🎯 Workflow Thực Tế

### Kịch bản 1: Tạo màn hình mới hoàn chỉnh
```
Bước 1: Tạo screen với Composer (⌘ + I)
"Tạo PendingOrdersScreen cho Manager approval với layout, 
 FlatList, loading state, error handling theo standards"

Bước 2: Tạo service và types
"Tạo pendingOrderService với API calls và types trong 
 pending-order.types.ts"

Bước 3: Add navigation
"Thêm PendingOrdersScreen vào MainTabNavigator với 
 tab icon và title"

Bước 4: Optimize
"Review và optimize PendingOrdersScreen: 
 - Add React.memo cho OrderCard
 - Add useMemo cho filtered data
 - Optimize FlatList với proper keyExtractor và getItemLayout"
```

### Kịch bản 2: Implement feature phức tạp
```
Bước 1: Planning với Chat
"Giải thích cách implement order cancellation workflow 
 từ Kitchen -> Manager -> Cashier theo rules"

Bước 2: Implement từng phần với Composer
"Tạo socket service với event handlers cho order cancellation"
"Tạo UI cho Kitchen staff cancel order"
"Tạo Manager approval screen"
"Tạo Cashier notification listener"

Bước 3: Testing và Refinement
"Review flow và add error handling cho mỗi bước"
"Add retry logic nếu socket disconnect"
"Add offline support với AsyncStorage queue"
```

### Kịch bản 3: Refactor existing code
```
Bước 1: Select code cần refactor
Bước 2: ⌘ + K và yêu cầu
"Refactor component này:
 - Extract business logic ra custom hook
 - Optimize với React.memo và useCallback
 - Add proper TypeScript types
 - Follow coding standards"
```

## 🔥 Pro Tips

### 1. **Sử dụng "@" để reference files**
```
"Cập nhật @MainTabNavigator.tsx để thêm OrdersScreen"
"Theo style của @LoginScreen.tsx, tạo RegisterScreen"
```

### 2. **Yêu cầu giải thích**
```
"Explain order cancellation workflow step by step"
"Tại sao nên dùng React Query thay vì useEffect + fetch?"
"So sánh Zustand vs Redux cho project này"
```

### 3. **Debug với context**
```
"Component này bị re-render nhiều lần, tìm nguyên nhân và fix"
"FlatList scroll lag khi có 1000 items, optimize"
"Token expire nhưng không auto refresh, debug"
```

### 4. **Yêu cầu multiple solutions**
```
"Đề xuất 3 cách implement real-time sync giữa Manager và Cashier,
 so sánh ưu nhược điểm"
```

### 5. **Incremental improvements**
```
"Review LoginScreen và suggest 5 improvements"
"Code review MainTabNavigator và list potential issues"
```
